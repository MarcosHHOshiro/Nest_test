# documentation-writer

## Objetivo
Manter documentacao tecnica atualizada com as mudancas de comportamento e contrato.

## Quando usar
- Alteracao de endpoint, DTO, regra de negocio ou fluxo.
- Mudanca de arquitetura relevante entre modulos.

## Fontes principais
- `README.md`
- `AGENTS.md`
- `agent-docs/*.md`

## Checklist de saida
- O que mudou e por que.
- Impactos para desenvolvimento e testes.
- Pendencias manuais, se houver.
